Getting Started

This guide walks you through the essential steps to start using Hexactyl. By the end, you will have a fully configured admin account, a game server running in Docker, and a clear understanding of the dashboard.

First Login

After completing the installation, open your browser and navigate to your panel URL. You will see the login page.

  1. Enter the email address you specified when creating the admin user.
  2. Enter the password you set during php artisan p:user:make.
  3. Click Sign In to access the admin dashboard.

If you forgot your credentials, you can reset the admin password by running:

php artisan p:user:make --email=admin@yourdomain.com

This will prompt you for a new password and confirm the account exists.

Understanding the Dashboard

The dashboard provides an at-a-glance view of your entire hosting environment:

  • Total Servers — The number of active game servers across all nodes.
  • Total Users — Registered users on the panel.
  • Nodes Online — Nodes currently connected and healthy.
  • Recent Activity — A feed of recent server creations, user registrations, and administrative actions.

The left sidebar in the admin area provides access to Servers, Users, Nodes, Locations, Server Eggs, and Settings. Each section is described in this guide.

Creating Your First Server

Before you can create a server, you need at least one node configured. Follow these steps:

  1. Navigate to Nodes in the admin sidebar and click Create New.
  2. Fill in the node name, FQDN, and the port range for the daemon. The default daemon port is 8080.
  3. Copy the Node Auto-Configuration Token from the node page. You will need this when setting up the Wings daemon on the server.
  4. Go to Servers and click Create New.
  5. Select the node, assign a server name, and choose a Nest (game category such as Minecraft, Rust, etc.).
  6. Select the Egg (specific game type) and configure the startup command, memory, disk, and CPU limits.
  7. Assign the server to a user and click Create Server.

The panel will queue the server creation task. Once the Wings daemon picks it up, the server will appear as Running in the dashboard.

Node Configuration Basics

Nodes are the physical machines that run your game servers. Each node runs the Wings daemon, which communicates with the panel over HTTPS.

Node Settings

  • Name — A human-readable identifier for the node (e.g., “US-East-1”).
  • FQDN — The fully qualified domain name that Wings will use to connect (e.g., node1.example.com).
  • Port Range — The range of ports available for game servers. Default is 25565–25655 for Minecraft.
  • Memory & Disk — The total allocatable resources on the node. These values are used to determine if new servers can be placed.

After creating the node, install Wings on the target machine and run the auto-configuration command provided on the node page.

User Management

Hexactyl includes a full user management system. Users can register through the panel or be created manually by an admin.

Creating Users

  1. Navigate to Users in the admin panel and click Create New.
  2. Enter the user’s email, first name, last name, and a password.
  3. Optionally, check Root Administrator to grant full admin access.
  4. Click Create User to finish.

Assigning Servers

When creating or editing a server, you can assign it to a specific user. That user will then be able to manage the server through their own dashboard, start/stop it, access the console, and upload files.

Permissions

Non-admin users can only see and manage servers that have been explicitly assigned to them. Admins have unrestricted access to all nodes, servers, users, and settings.

API Key Setup

Hexactyl provides a RESTful API for building custom integrations. To use the API, you need to generate an API key pair.

Generating Keys

  1. Log in to the panel and navigate to Account Settings.
  2. Click the API Credentials tab.
  3. Click Create New and provide a descriptive label (e.g., “CI/CD Pipeline”).
  4. Copy the API Key and API Secret immediately. The secret is only shown once.

Using the API

All API requests require two headers:

Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

For client API keys, include the key and secret in the request body or as query parameters. The API base URL is:

https://yourpanel.example.com/api/client

Refer to the API Reference page for a full list of endpoints and examples.
