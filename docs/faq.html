Frequently Asked Questions

This page answers common questions about installing, configuring, and troubleshooting Hexactyl.

Installation

1. What operating systems are supported?

Hexactyl supports Ubuntu 20.04+, Debian 11+, and CentOS 8+. Ubuntu 22.04 LTS is recommended for the best experience. Other distributions may work but are not officially tested.

2. Can I install Hexactyl on the same machine as my game servers?

Yes. The panel and the Wings daemon can run on the same machine. However, for production environments with more than a few servers, it is recommended to separate the panel onto its own machine for better security and performance.

3. Do I need Docker installed on the panel machine?

Docker is required on every machine that runs game servers (nodes). If the panel and game servers are on the same machine, Docker must be installed there. If the panel is on a separate machine, Docker is only needed on the nodes.

4. How do I update Hexactyl after installation?

Pull the latest code, install dependencies, rebuild assets, and run migrations:

cd /var/www/hexactyl
git pull origin main
composer install --no-dev --optimize-autoloader
yarn install && yarn build:production
php artisan migrate --force
sudo systemctl restart hexactyl-queue

Always back up your database before running migrations on a production panel.

5. Can I use MariaDB instead of MySQL?

Yes. MariaDB 10.2+ is fully supported and works identically to MySQL. In most cases, MariaDB is the default database on Debian-based systems.

Configuration

6. How do I change the panel URL?

Edit the APP_URL variable in your .env file and clear caches:

APP_URL=https://newdomain.com
php artisan config:clear
php artisan cache:clear

Also update the URL in the admin panel under Settings → General.

7. How do I enable HTTPS?

Install Certbot and obtain a certificate:

sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d panel.yourdomain.com

Update APP_URL in .env to use https://. Nginx will automatically redirect HTTP to HTTPS.

8. Can I use a different mail provider than Mailgun?

Yes. Hexactyl supports any SMTP server, Mailgun, Amazon SES, and Postmark. Set the MAIL_MAILER variable to smtp, mailgun, ses, or postmark and configure the corresponding settings in .env.

9. How do I set up automatic backups?

Configure S3-compatible storage in your .env file (see the Configuration page), then enable backups in Admin Panel → Settings → Backups. Users can then create manual backups or set schedules from their server management page.

10. How do I reset the admin password?

Run the user creation command with the admin email address:

php artisan p:user:make --email=admin@yourdomain.com

This will prompt you to set a new password for the account.

Nodes and Wings

11. My node shows as “Offline”. What should I check?

Verify the following:

  • The Wings daemon is running: sudo systemctl status wings
  • The firewall allows port 8080 (daemon) and the game server port range.
  • The panel URL in /etc/wings/config.yml matches your APP_URL.
  • The TLS certificate on the panel is valid and not expired.
  • Check Wings logs: sudo journalctl -u wings -f

12. How do I add a second node?

In the admin panel, go to Nodes → Create New. Fill in the details, copy the auto-configuration token, and run the Wings setup command on the new machine. The process is identical to the first node.

13. Can I move a server from one node to another?

Hexactyl does not currently support live server migration between nodes. You would need to manually transfer the server files using rsync or SCP, then update the server record in the database to point to the new node.

14. How do I update Wings?

sudo systemctl stop wings
curl -L -o /usr/local/bin/wings "https://github.com/Hexactyl-Projects/wings/releases/latest/download/wings_linux_amd64"
chmod u+x /usr/local/bin/wings
sudo systemctl start wings

Wings is always compatible with the latest panel version, so update both when possible.

Troubleshooting

15. The panel shows a blank page or 500 error.

Check the Laravel log at storage/logs/laravel.log for detailed error messages. Common causes include:

  • Missing or incorrect database credentials in .env.
  • Redis not running.
  • Incorrect file permissions on storage/ and bootstrap/cache/.
  • The APP_KEY is missing or invalid.

Run php artisan config:clear and php artisan cache:clear to rule out cached configuration issues.

16. Server creation fails with “Docker error”.

Ensure Docker is installed and running on the target node. Verify that the Wings user is in the docker group. Check that the Docker image specified in the egg exists and is pullable. Run docker pull hexactyl/yellow:latest manually to test.

17. How do I check if the queue worker is running?

sudo systemctl status hexactyl-queue
php artisan queue:status

If the worker is not running, start it with sudo systemctl start hexactyl-queue. Check storage/logs/laravel.log for queue-related errors.

18. My users are not receiving emails.

Test your mail configuration:

php artisan mail:test youremail@example.com

If emails are not received, check your SMTP credentials, ensure port 587 (or your configured port) is open, and verify that your mail server is not rate-limiting or blocking the panel’s IP address.

19. How do I enable debug mode to see detailed errors?

Set the following in your .env file (never do this in production):

APP_DEBUG=true
APP_ENV=local

Clear caches with php artisan config:clear. Debug mode will display full error pages with stack traces. Disable it immediately after troubleshooting.

20. Where can I get help if none of these solutions work?

Visit the GitHub repository and open an issue with a detailed description of your problem. Include your PHP version, OS, Wings version, and relevant log excerpts. The community and maintainers are active and responsive.
