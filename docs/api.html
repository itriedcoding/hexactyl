API Reference

Hexactyl provides a full REST API for building custom integrations, bots, and automation workflows. All endpoints require authentication using API key pairs generated through the panel.

Authentication

Hexactyl uses API key pairs for authentication. Each key consists of a public key and a secret key. The public key identifies the key; the secret key is used to sign requests.

Generating API Keys

  1. Log in to the panel and go to Account Settings → API Credentials.
  2. Click Create New and enter a label for the key.
  3. Copy both the API Key and API Secret. The secret is only shown once.

Making Authenticated Requests

All API requests must include the following headers:

Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
Accept: Application/vnd.hexactyl.v1+json

The API base URL is:

https://yourpanel.example.com/api/client

For application-level (admin) endpoints, use the /api/application prefix and include the key in the X-Auth-Key header.

Rate Limiting

The API enforces rate limits to prevent abuse. The current limits are:

Request Type Limit Window
Authenticated requests60Per minute
Unauthenticated requests30Per minute
Server power actions10Per minute
File uploads20Per minute

Rate limit headers are included in every API response:

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 58
X-RateLimit-Reset: 1711000060

If you exceed the limit, the API returns 429 Too Many Requests. Wait for the reset window before retrying.

Error Handling

The API returns standard HTTP status codes and JSON error bodies. All errors follow this structure:

{
    "errors": [
        {
            "code": "ValidationException",
            "status": "422",
            "detail": "The given data was invalid.",
            "meta": {
                "field": "email",
                "reason": "The email has already been taken."
            }
        }
    ]
}

Common Status Codes

Code Meaning
200Request succeeded.
201Resource created successfully.
204Request succeeded with no response body.
400Malformed request. Check the request body or parameters.
401Authentication failed. Verify your API key and secret.
403Insufficient permissions for this action.
404Resource not found.
422Validation error. Check the meta field for details.
429Rate limit exceeded. Retry after the reset window.
500Internal server error. Contact the panel administrator.

Always implement retry logic with exponential backoff for 429 and 500 errors.

Versioning

The API is versioned via the Accept header. The current version is v1:

Accept: Application/vnd.hexactyl.v1+json

New versions will be introduced as breaking changes are made. The previous version will be supported for at least 12 months after a new version is released.
