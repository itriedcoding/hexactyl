Installation

This guide covers the full installation process for Hexactyl on a Linux server. Both a manual installation and a Docker-based installation are provided.

System Requirements

Ensure your server meets the following minimum requirements before proceeding:

Step-by-Step Installation

Follow each step in order. All commands are for Ubuntu/Debian. Adjust package manager commands for other distributions.

Step 1: Update System Packages

sudo apt update && sudo apt upgrade -y

Step 2: Install PHP and Required Extensions

sudo apt install -y php8.2 php8.2-{cli,common,gd,mbstring,bcmath,xml,fpm,curl,zip,mysql,intl,tokenizer,dom,redis,imagick}

Verify the installation:

php -v

Step 3: Install MySQL or MariaDB

sudo apt install -y mysql-server
sudo mysql_secure_installation

Create the database and user:

sudo mysql -u root -p
CREATE DATABASE hexactyl;
CREATE USER 'hexactyl'@'127.0.0.1' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON hexactyl.* TO 'hexactyl'@'127.0.0.1';
FLUSH PRIVILEGES;
EXIT;

Step 4: Install Redis

sudo apt install -y redis-server
sudo systemctl enable --now redis-server

Step 5: Install Docker

curl -fsSL https://get.docker.com | sudo bash
sudo usermod -aG docker $USER
sudo systemctl enable --now docker

Log out and back in for group changes to take effect.

Step 6: Install Node.js

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v && npm -v

Step 7: Install Composer

curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

Step 8: Clone and Configure the Panel

cd /var/www
sudo git clone https://github.com/Hexactyl-Projects/hexactyl.git
cd hexactyl
sudo cp .env.example .env
sudo composer install --no-dev --optimize-autoloader

Generate the application key:

sudo php artisan key:generate --force

Step 9: Configure the .env File

Edit /var/www/hexactyl/.env and set the following values:

APP_URL=https://panel.yourdomain.com
APP_SERVICE=hexactyl-panel

DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=hexactyl
DB_USERNAME=hexactyl
DB_PASSWORD=your_secure_password

CACHE_DRIVER=redis
SESSION_DRIVER=redis
QUEUE_DRIVER=redis

REDIS_HOST=127.0.0.1
REDIS_PORT=6379

Step 10: Run Database Migrations

sudo php artisan migrate --force
sudo php artisan db:seed --force

Step 11: Set File Permissions

sudo chown -R www-data:www-data /var/www/hexactyl
sudo chmod -R 755 /var/www/hexactyl/storage
sudo chmod -R 755 /var/www/hexactyl/bootstrap/cache

Step 12: Build Frontend Assets

yarn install
yarn build:production

Step 13: Set Up Queue Worker

Create a systemd service file at /etc/systemd/system/hexactyl-queue.service:

[Unit]
Description=Hexactyl Queue Worker
After=redis.service

[Service]
User=www-data
Group=www-data
Restart=always
ExecStart=/usr/bin/php /var/www/hexactyl/artisan queue:work --queue=high,standard,low --sleep=3 --tries=3
StartLimitInterval=180
StartLimitBurst=30
RestartSec=5s

[Install]
WantedBy=multi-user.target
sudo systemctl enable --now hexactyl-queue

Step 14: Configure Cron Jobs

Add the following to sudo crontab -e:

* * * * * php /var/www/hexactyl/artisan schedule:run >> /dev/null 2>&1

Docker Installation

Hexactyl ships with a docker-compose.yml for quick setup. This method uses containers for all services.

git clone https://github.com/Hexactyl-Projects/hexactyl.git
cd hexactyl
cp .env.example .env
docker compose up -d
docker compose exec panel php artisan key:generate --force
docker compose exec panel php artisan migrate --force
docker compose exec panel php artisan db:seed --force

The Docker Compose stack includes Nginx, PHP-FPM, MySQL, Redis, and a queue worker container. All services are automatically linked.

Web Server Configuration (Nginx)

Create a virtual host file at /etc/nginx/sites-available/hexactyl.conf:

server {
    listen 80;
    server_name panel.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name panel.yourdomain.com;

    root /var/www/hexactyl/public;
    index index.php index.html;

    ssl_certificate /etc/letsencrypt/live/panel.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/panel.yourdomain.com/privkey.pem;

    client_max_body_size 100M;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.ht {
        deny all;
    }
}
sudo ln -s /etc/nginx/sites-available/hexactyl.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

SSL/TLS Setup with Let’s Encrypt

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d panel.yourdomain.com

Certbot will automatically configure Nginx with SSL and set up auto-renewal. Verify renewal works:

sudo certbot renew --dry-run

Install the Wings Daemon

On each game server node, install Wings to run Docker containers:

curl -L -o /usr/local/bin/wings "https://github.com/Hexactyl-Projects/wings/releases/latest/download/wings_linux_amd64"
chmod u+x /usr/local/bin/wings

Copy the auto-configuration command from the node page in the admin panel and run it as root:

sudo wings configure --panel-url https://panel.yourdomain.com --token YOUR_NODE_TOKEN --auto

Create a systemd service for Wings at /etc/systemd/system/wings.service:

[Unit]
Description=Hexactyl Wings Daemon
After=docker.service
Requires=docker.service

[Service]
User=root
WorkingDirectory=/etc
ExecStart=/usr/local/bin/wings
Restart=on-failure
StartLimitInterval=180
StartLimitBurst=30
RestartSec=5s

[Install]
WantedBy=multi-user.target
sudo systemctl enable --now wings

Verify Installation

Open your browser and visit https://panel.yourdomain.com. You should see the login page. Log in with the admin credentials you created and verify that the dashboard loads correctly. Add a node and create a test server to confirm everything is working end-to-end.
