Server Templates

Server templates in Hexactyl allow administrators to define pre-configured server blueprints. Instead of manually entering startup commands, environment variables, and resource limits every time a new server is created, a template encapsulates all of these settings in a reusable package.

What Are Server Templates?

A server template is a saved configuration that includes:

  • Nest and Egg — The game type (e.g., Minecraft Paper, Rust, CS2).
  • Startup Command — The default command used to launch the server.
  • Default Environment Variables — Pre-filled variables such as SERVER_MEMORY, MAX_PLAYERS, or MOTD.
  • Resource Limits — Default memory, disk, CPU, and I/O allocations.
  • Docker Image — The container image used to run the server.
  • Allocations — Default port allocation rules.

Templates are stored in the database and are available to all administrators. They can be created, edited, duplicated, or deleted at any time.

Creating a Template

To create a new server template:

  1. Navigate to Admin Panel → Server Templates.
  2. Click Create New Template.
  3. Provide a Template Name (e.g., “Minecraft Survival”).
  4. Select the Nest and Egg this template applies to.
  5. Configure the default startup command. You can use variable placeholders enclosed in curly braces (e.g., {SERVER_MEMORY}).
  6. Set default values for each environment variable. Mark required variables as mandatory.
  7. Define resource defaults: memory, disk, CPU, and I/O limits.
  8. Select the default Docker image for this template.
  9. Click Create Template to save.

Once created, the template appears in the templates list and is available during server creation.

Using Templates to Deploy Servers

When creating a new server, administrators can select a template to pre-fill all configuration fields:

  1. Navigate to Servers → Create New.
  2. In the Template dropdown, select the desired template.
  3. All fields (startup command, environment variables, resource limits, Docker image) will be populated automatically.
  4. Adjust any values as needed for this specific server instance.
  5. Select the target node, assign a user, and complete the server creation.

Templates only provide defaults. You can override any field before creating the server.

Example Workflow

# Template: "Minecraft Survival"
Nest: Minecraft
Egg: Paper
Startup: java -Xms{SERVER_MEMORY}M -Xmx{SERVER_MEMORY}M -jar server.jar nogui
Default Memory: 2048 MB
Default Disk: 10240 MB
Default Image: hexactyl/yellow:latest

# When creating a server:
1. Select "Minecraft Survival" template
2. Server name: "Survival Server #1"
3. User: john@example.com
4. Node: US-East-1
5. Click Create

The server will be created with all template settings applied.

Managing Templates

The templates list provides options to manage your saved templates:

  • Edit — Modify any field of an existing template. Changes only affect new servers; existing servers are not modified.
  • Duplicate — Create a copy of a template with a new name. Useful for creating variations of an existing setup.
  • Delete — Remove a template. This does not affect servers created from that template.
  • Reorder — Drag and drop templates to change their display order in the dropdown.

Templates are stored in the server_templates database table. You can also manage them via the API.

Template API

Server templates can be managed programmatically through the Hexactyl API. This is useful for automating template creation from external tools or CI/CD pipelines.

List Templates

GET /api/client/templates

Response:
{
    "data": [
        {
            "id": 1,
            "name": "Minecraft Survival",
            "nest_id": 1,
            "egg_id": 5,
            "startup": "java -Xms{SERVER_MEMORY}M -Xmx{SERVER_MEMORY}M -jar server.jar nogui",
            "docker_image": "hexactyl/yellow:latest",
            "memory": 2048,
            "disk": 10240,
            "created_at": "2026-01-15T10:30:00Z"
        }
    ]
}

Create Template

POST /api/client/templates

{
    "name": "Rust Server",
    "nest_id": 2,
    "egg_id": 12,
    "startup": "./rust_server -batchmode +server.port {SERVER_PORT} +server.identity {SERVER_IDENTITY}",
    "docker_image": "hexactyl/rust:latest",
    "memory": 4096,
    "disk": 20480,
    "environment": {
        "SERVER_PORT": "28015",
        "SERVER_IDENTITY": "my_server",
        "SERVER_NAME": "My Rust Server"
    }
}

Response:
{
    "data": {
        "id": 3,
        "name": "Rust Server",
        "nest_id": 2,
        "egg_id": 12,
        "created_at": "2026-03-20T14:00:00Z"
    }
}

Delete Template

DELETE /api/client/templates/{id}

Response: 204 No Content

Refer to the API Reference page for full authentication details and additional endpoints.
